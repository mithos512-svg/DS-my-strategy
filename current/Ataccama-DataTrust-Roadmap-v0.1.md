# Ataccama Data Trust System — Roadmap 2026–2028
**Статус:** черновик v0.1  
**Дата:** 2026-07-01  
**Автор:** Product Owner  
**Аудитория:** C-level + IT руководство + Data команда  

---

## Контекст и проблема

**Индустрия:** Food Production  
**Масштаб данных:** 100–10 000 источников  

**Две горящие проблемы:**
1. **Недоверие бизнеса к данным** — решения принимаются "на глаз", отчёты расходятся, бизнес перепроверяет данные вручную
2. **Нет visibility** — непонятно что где лежит, кто владелец, откуда цифра в отчёте

**Текущее состояние Ataccama:**
- Catalog — BAU (работает в производственном режиме)
- DQ + Lineage — deployment (в процессе внедрения)
- MDM — не в скоупе

---

## Цель: Data Trust System

> Данные, которым доверяет бизнес — потому что видит их качество, происхождение и владельца.

**Измеримый результат через 2-3 года:**
- % бизнес-решений, принятых на основе сертифицированных данных: >80%
- Время поиска данных для аналитика: с дней до часов
- Покрытие критических доменов DQ-мониторингом: >90%

## Roadmap 2026–2028

### Фаза 1 — Foundation (2026 Q3–Q4)
**Тема:** "Знаем что у нас есть"

| # | Инициатива | Модуль | Результат |
|---|-----------|--------|-----------|
| 1.1 | Завершить DQ deployment на критических доменах (продукт, поставщик, рецептура) | DQ | DQ-мониторинг работает на ТОП-10 доменах |
| 1.2 | Завершить Lineage для ключевых отчётов | Lineage | Видна цепочка от источника до отчёта |
| 1.3 | Data Ownership модель — назначить стюардов | Catalog | Каждый домен имеет владельца |
| 1.4 | Data Trust Dashboard v1 | DQ + Catalog | Бизнес видит качество данных в реальном времени |

**Milestone:** первый сертифицированный датасет — продуктовый каталог.

---

### Фаза 2 — Expansion (2027 Q1–Q2)
**Тема:** "Доверяем тому что используем"

| # | Инициатива | Модуль | Результат |
|---|-----------|--------|-----------|
| 2.1 | Расширить DQ на 50+ доменов | DQ | Системный охват |
| 2.2 | Self-service каталог для бизнеса | Catalog | Аналитик находит данные без обращения в IT |
| 2.3 | Автоматическая классификация данных | Catalog + DQ | Снижение ручного труда стюардов |
| 2.4 | Data Quality SLA — договорённости с бизнесом | Process | Формальные ожидания по качеству |

**Milestone:** 50% аналитических запросов закрываются через Catalog без IT-посредника.

---

### Фаза 3 — Trust at Scale (2027 Q3 — 2028 Q4)
**Тема:** "Качество данных — конкурентное преимущество"

| # | Инициатива | Модуль | Результат |
|---|-----------|--------|-----------|
| 3.1 | Predictive DQ — предсказание деградации качества | DQ + AI | Проблемы выявляются до попадания в отчёты |
| 3.2 | Полный end-to-end Lineage | Lineage | От сырья до KPI — прослеживаемость 100% |
| 3.3 | Data Marketplace — внутренний рынок данных | Catalog | Команды публикуют и потребляют данные как продукты |
| 3.4 | Data Trust Score — единая метрика доверия | DQ + Catalog | Один показатель здоровья данных для C-level |

**Milestone:** Data Trust Score введён как KPI наравне с финансовыми показателями.

## Риски и зависимости

| Риск | Вероятность | Митигация |
|------|-------------|-----------|
| DQ deployment затягивается | Высокая | Зафиксировать скоуп Фазы 1, не расширять до завершения |
| Бизнес не назначает стюардов | Средняя | Спонсорство C-level, включить в KPI менеджеров |
| Низкое adoption Catalog | Средняя | UX-фокус, обучение, champions в бизнес-командах |
| Рост источников данных опережает покрытие | Высокая | Приоритизация по бизнес-критичности, не по объёму |
