# Дневник обучения - 20 июля 2026

## FPF-2 - начало просмотра (2026-07-20)

**Заметка - кейс-менеджмент: колонка "предмет кейса" + шаблонизация:**

Идея: в системе кейс-менеджмента нужно добавить колонку **"предмет кейса"** -
явно фиксировать, о чём кейс, а не только его статус/приоритет/категорию.

**Развитие идеи - community templates:**
Если несколько кейсов повторяют похожий предмет/паттерн - имеет смысл выделить это
в **шаблон (community template)**, который можно переиспользовать для похожих кейсов
в будущем, а не решать каждый раз с нуля.

**Связь с предыдущими заметками:**
- Трекер команды (15.07) - там уже есть атрибуты категория+приоритет; "предмет кейса"
  может стать третьим ключевым атрибутом
- Декомпозиция сложности (15.07) - шаблон для похожих кейсов - это готовая
  декомпозиция, которую не нужно строить заново
- FPF: "что открывает следующий ход" (16.07) - выявление шаблона открывает
  возможность ускорить решение будущих похожих кейсов

**Практическое применение:** добавить в трекер команды (W29 РП3, первая сессия уже
прошла) атрибут "предмет кейса" при следующей итерации настройки трекера. Подумать,
какие повторяющиеся паттерны кейсов уже видны в текущей практике - оформить как
первые community templates.

**Статус: идея для доработки трекера команды, продолжаю смотреть FPF-2.**
