# Дневник обучения — 6 июля 2026

## IWE развитие — Монофокус, §6.04 (2026-07-06)

**Заметка — ограничение применения:**
§6.04 хорошая концепция (2-4 фокусных задачи в день, не больше), но полное применение
подразумевает ведение рабочих проектов через IWE — что пока невозможно из-за корпоративной
политики запрета Claude на рабочих задачах Ataccama.

**Статус:** идея понравилась, внедрение отложено до решения вопроса с политикой.

**Открытый вопрос:** как применить монофокус в личном треке (S1/S2/IWE) без рабочих проектов?

**Идеи для решения корпоративного ограничения (от Claude):**

1. **Разделить личное и рабочее явно** — IWE ведёт только личные треки (S1/S2, здоровье, саморазвитие),
   рабочие проекты Ataccama остаются на рабочем ПК. Монофокус применять только к личным РП —
   этого уже достаточно для начала.

2. **Анонимизированный контекст** — обсуждать рабочие задачи без упоминания компании, продуктов,
   клиентов и коллег (только структура и методология). Но это серая зона, не рекомендую без
   явного согласования с IT/Legal.

3. **Локальная LLM** — развернуть локальную модель (Ollama + Llama/Mistral) на рабочем ПК для
   задач, которые нельзя выносить в облако. Сложнее технически, зато без ограничений политики.

4. **Согласовать исключение** — написать запрос в IT/Legal компании с обоснованием
   (персональный инструмент саморазвития, не обработка корпоративных данных). Возможно,
   Claude.ai как личный инструмент допустим при условии что рабочие данные не вводятся.

5. **Подождать** — многие компании пересматривают AI-политику каждые 6-12 месяцев.

**Статус:** открытый вопрос, без спешки. Вернуться на Strategy Session когда появится идея.
